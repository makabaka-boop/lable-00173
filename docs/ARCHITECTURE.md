## 系统总体架构

### 1.1 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                     客户端层 (Client Layer)                  │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Web 浏览器 (Vue 3 + TypeScript + Element Plus)      │  │
│  │  ┌─────────────┬─────────────┬─────────────┐        │  │
│  │  │ 行情中心    │ 交易管理    │ 持仓管理    │        │  │
│  │  └─────────────┴─────────────┴─────────────┘        │  │
│  │  ┌─────────────┬─────────────┐                      │  │
│  │  │ 策略回测    │ 个人资料    │                      │  │
│  │  └─────────────┴─────────────┘                      │  │
│  └──────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                          ↓ HTTP/REST
┌─────────────────────────────────────────────────────────────┐
│                   API 网关层 (API Gateway)                   │
│  ├─ 请求路由                                               │
│  ├─ 认证验证                                               │
│  ├─ 速率限制                                               │
│  └─ 日志记录                                               │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                  业务逻辑层 (Business Logic)                 │
│  ┌──────────────┬──────────────┬──────────────┐            │
│  │ 认证服务     │ 行情服务     │ 交易服务     │            │
│  └──────────────┴──────────────┴──────────────┘            │
│  ┌──────────────┬──────────────┬──────────────┐            │
│  │ 持仓服务     │ 回测服务     │ 用户服务     │            │
│  └──────────────┴──────────────┴──────────────┘            │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                   数据访问层 (Data Access)                   │
│  ├─ ORM (SQLAlchemy)                                       │
│  ├─ 数据验证                                               │
│  └─ 缓存管理                                               │
└─────────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────────┐
│                    数据层 (Data Layer)                       │
│  ├─ SQLite 数据库                                          │
│  ├─ 数据备份                                               │
│  └─ 数据恢复                                               │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 架构特点

- **分层架构**：清晰的层次划分，便于维护和扩展
- **前后端分离**：独立开发和部署
- **RESTful API**：标准的 HTTP 接口
- **微服务思想**：模块化的业务逻辑
- **容器化部署**：使用 Docker 实现一致的运行环境

## 接口设计

### 4.1 API 设计原则

- **RESTful 风格**：使用 HTTP 方法表示操作
- **版本管理**：支持 API 版本控制
- **统一响应格式**：所有接口返回统一格式
- **错误处理**：统一的错误响应
- **文档完整**：每个接口都有详细文档

### 4.2 响应格式

**成功响应**：
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    // 具体数据
  }
}
```

**错误响应**：
```json
{
  "code": 400,
  "message": "错误描述",
  "data": null
}
```

## 数据流程

### 5.1 用户登录数据流

```
用户输入凭证
    ↓
前端发送 POST /api/auth/login
    ↓
后端验证凭证
    ├─ 查询数据库
    ├─ 比对密码
    └─ 生成 Token
    ↓
返回用户信息和 Token
    ↓
前端保存 Token 到 localStorage
    ↓
前端设置 Authorization header
    ↓
跳转到首页
```

### 5.2 交易执行数据流

```
用户创建订单
    ↓
前端验证参数
    ↓
前端发送 POST /api/trading/orders
    ↓
后端验证参数
    ├─ 检查持仓数量（卖出）
    └─ 保存订单到数据库
    ↓
返回订单信息
    ↓
用户执行订单
    ↓
前端发送 PUT /api/trading/orders/{id}/execute
    ↓
后端执行订单
    ├─ 更新持仓信息
    ├─ 生成交易记录
    └─ 更新订单状态
    ↓
返回执行结果
    ↓
前端刷新数据
```

### 5.3 持仓更新数据流

```
订单执行
    ↓
后端查询现有持仓
    ├─ 存在 → 更新数量和成本价
    └─ 不存在 → 创建新持仓
    ↓
计算新的成本价
    ├─ 买入：(旧成本 × 旧数量 + 新价格 × 新数量) / (旧数量 + 新数量)
    └─ 卖出：保持不变
    ↓
更新数据库
    ↓
前端查询持仓列表
    ↓
计算收益和收益率
    ├─ 市值 = 数量 × 现价
    ├─ 收益 = 市值 - 成本
    └─ 收益率 = 收益 / 成本 × 100%
    ↓
显示持仓信息
```

### 5.4 回测执行数据流

```
用户创建回测任务
    ↓
前端验证参数
    ↓
前端发送 POST /api/backtest/run
    ↓
后端验证参数
    ↓
获取历史数据
    ├─ 查询 K 线数据
    └─ 按时间排序
    ↓
应用策略规则
    ├─ 计算技术指标
    ├─ 生成交易信号
    └─ 模拟交易执行
    ↓
计算性能指标
    ├─ 总收益 = 最终资金 - 初始资金
    ├─ 收益率 = 总收益 / 初始资金 × 100%
    ├─ 最大回撤 = 最大值 - 最小值 / 最大值
    ├─ 夏普比率 = 平均收益 / 标准差
    └─ 胜率 = 盈利交易数 / 总交易数 × 100%
    ↓
保存回测结果
    ↓
返回结果
    ↓
前端显示回测报告
```

## 安全策略

### 6.1 身份认证

- **认证方式**：用户名 + 密码
- **Token 机制**：JWT Token
- **Token 有效期**：24 小时
- **刷新机制**：支持 Token 刷新

### 6.2 授权控制

- **基于角色的访问控制 (RBAC)**
- **权限检查**：每个接口都进行权限验证
- **资源隔离**：用户只能访问自己的数据

### 6.3 数据安全

- **传输安全**：HTTPS 加密传输
- **存储安全**：密码使用加密存储（实际应用中）
- **SQL 注入防护**：使用参数化查询
- **XSS 防护**：输入验证和输出编码

### 6.4 会话管理

- **会话超时**：30 分钟无操作自动登出
- **并发控制**：同一用户只能有一个活跃会话
- **登出处理**：清除 Token 和会话数据

### 6.5 审计日志

- **操作日志**：记录所有重要操作
- **访问日志**：记录 API 访问
- **错误日志**：记录系统错误
- **日志保留**：保留 90 天

## 性能优化

### 7.1 前端优化

- **代码分割**：按需加载模块
- **懒加载**：延迟加载组件
- **缓存策略**：使用浏览器缓存
- **压缩**：Gzip 压缩资源
- **CDN**：使用 CDN 加速

### 7.2 后端优化

- **数据库索引**：为常用查询字段建立索引
- **查询优化**：避免 N+1 查询
- **缓存**：使用内存缓存热数据
- **连接池**：数据库连接池管理
- **异步处理**：长时间操作异步执行

### 7.3 网络优化

- **请求合并**：减少 HTTP 请求数
- **分页加载**：大数据集分页处理
- **增量更新**：只传输变化的数据
- **压缩**：启用 Gzip 压缩

### 7.4 性能指标

| 指标 | 目标 |
|------|------|
| 首屏加载时间 | < 3 秒 |
| API 响应时间 | < 1 秒 |
| 数据库查询时间 | < 500 ms |
| 页面交互响应 | < 100 ms |
| 并发用户数 | 1000+ |

```

├── backend service
│   ├── Python 3.11
```

```
┌─────────────────────────────────────┐
│      Docker Compose Network         │
│  ┌──────────────┬──────────────┐   │

│  │   :8080      │   :8000      │   │
│  └──────────────┴──────────────┘   │
│         ↓              ↓             │
│    Nginx Proxy    Flask App         │
```

```bash
# 查看日志

docker-compose down

# 重启服务
docker-compose restart
```

## 监控与告警

### 10.1 监控指标

- **系统指标**：CPU、内存、磁盘使用率
- **应用指标**：请求数、响应时间、错误率
- **数据库指标**：查询时间、连接数、锁等待
- **业务指标**：用户数、交易量、回测数

### 10.2 告警规则

- **高 CPU 使用率**：> 80%
- **高内存使用率**：> 85%
