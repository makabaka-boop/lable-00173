## 产品概述

### 1.1 产品定义

证券交易系统是一个功能完整的在线股票交易平台，为投资者提供实时行情查看、交易执行、持仓管理和策略回测等核心功能。系统采用前后端分离架构，支持跨平台访问。

### 1.2 产品目标

- 为投资者提供便捷的股票交易体验
- 实现完整的交易流程管理
- 支持策略回测和性能分析
- 提供实时的行情数据和交易信息

### 1.3 产品范围

**包含内容**：
- 用户认证与管理
- 实时行情展示
- 交易订单管理
- 持仓跟踪
- 策略回测
- 个人资料管理

**不包含内容**：
- 真实资金交易
- 第三方支付集成
- 高频交易功能
- 融资融券

## 功能需求

### 2.1 用户认证模块

#### 2.1.1 用户登录
- **功能描述**：用户使用用户名和密码登录系统
- **前置条件**：用户已注册
- **主流程**：
  1. 用户输入用户名和密码
  2. 系统验证凭证
  3. 验证成功，返回 token 和用户信息
  4. 前端保存 token 到 localStorage
- **异常处理**：
  - 用户名或密码错误：返回错误提示
  - 账户被锁定：返回锁定提示
- **性能要求**：响应时间 < 2 秒

#### 2.1.2 用户登出
- **功能描述**：用户退出系统
- **主流程**：
  1. 用户点击登出按钮
  2. 清除本地 token
  3. 重定向到登录页面

#### 2.1.3 会话管理
- **功能描述**：管理用户会话状态
- **需求**：
  - Token 有效期：24 小时
  - 页面刷新时自动恢复登录状态
  - 无操作 30 分钟后自动登出

### 2.2 行情中心模块

#### 2.2.1 股票列表展示
- **功能描述**：展示所有可交易的股票信息
- **显示字段**：
  - 股票代码
  - 股票名称
  - 当前价格
  - 涨跌幅
  - 涨跌额
  - 最高价
  - 最低价
  - 成交量
  - 市盈率
- **功能需求**：
  - 支持按涨幅排序
  - 支持按成交量排序
  - 支持搜索功能
  - 实时数据更新（每 5 秒）

#### 2.2.2 K线图展示
- **功能描述**：展示股票的 K 线走势
- **需求**：
  - 支持日线、周线、月线切换
  - 显示开盘价、收盘价、最高价、最低价
  - 显示成交量
  - 显示平均值线
  - 支持放大缩小
  - 支持鼠标悬停显示详细数据

#### 2.2.3 股票详情
- **功能描述**：展示单个股票的详细信息
- **显示内容**：
  - 基本信息（代码、名称、价格）
  - K 线图
  - 技术指标
  - 相关操作按钮

### 2.3 交易管理模块

#### 2.3.1 订单创建
- **功能描述**：用户创建买入或卖出订单
- **输入参数**：
  - 股票代码
  - 交易类型（买入/卖出）
  - 数量
  - 价格
- **业务规则**：
  - 数量必须为正整数
  - 价格必须大于 0
  - 卖出时持仓数量必须充足
- **返回结果**：订单 ID 和状态

#### 2.3.2 订单管理
- **功能描述**：管理已创建的订单
- **订单状态**：
  - 待处理（pending）
  - 已成交（completed）
  - 已撤销（cancelled）
- **操作**：
  - 查看订单列表
  - 执行订单
  - 撤销订单
  - 查看订单详情

#### 2.3.3 订单执行
- **功能描述**：执行待处理的订单
- **执行流程**：
  1. 验证订单有效性
  2. 更新账户持仓
  3. 生成交易记录
  4. 更新订单状态为已成交
- **自动更新**：
  - 买入订单：增加持仓，减少余额
  - 卖出订单：减少持仓，增加余额

#### 2.3.4 交易历史
- **功能描述**：查看历史交易记录
- **显示字段**：
  - 交易代码
  - 交易类型
  - 数量
  - 价格
  - 总金额
  - 交易时间
  - 收益（仅卖出）

### 2.4 持仓管理模块

#### 2.4.1 持仓查看
- **功能描述**：查看当前持仓情况
- **显示信息**：
  - 股票代码和名称
  - 持仓数量
  - 成本价
  - 现价
  - 市值
  - 收益
  - 收益率
- **统计信息**：
  - 总市值
  - 总收益
  - 收益率

#### 2.4.2 持仓调整
- **功能描述**：调整持仓数量
- **操作**：
  - 加仓：增加持仓数量
  - 减仓：减少持仓数量
- **业务规则**：
  - 加仓数量必须为正整数
  - 减仓数量不能超过当前持仓
  - 操作后自动更新成本价

#### 2.4.3 持仓统计
- **功能描述**：统计持仓数据
- **统计指标**：
  - 总市值 = 所有持仓的市值之和
  - 总收益 = 总市值 - 总成本
  - 收益率 = 总收益 / 总成本 × 100%

### 2.5 策略回测模块

#### 2.5.1 回测创建
- **功能描述**：创建新的回测任务
- **输入参数**：
  - 回测名称
  - 股票代码
  - 策略类型（均线/RSI/MACD）
  - 开始日期
  - 结束日期
  - 初始资金
  - 策略参数
- **验证规则**：
  - 名称不能为空
  - 日期范围有效
  - 初始资金 > 0

#### 2.5.2 回测执行
- **功能描述**：执行回测任务
- **执行流程**：
  1. 获取历史数据
  2. 应用策略规则
  3. 模拟交易执行
  4. 计算性能指标
  5. 生成回测报告
- **性能要求**：
  - 1 年数据回测 < 10 秒
  - 5 年数据回测 < 30 秒

#### 2.5.3 回测结果
- **功能描述**：查看回测结果
- **显示指标**：
  - 初始资金
  - 最终资金
  - 总收益
  - 收益率
  - 最大回撤
  - 夏普比率
  - 胜率
  - 交易次数
  - 盈利交易数
  - 亏损交易数
- **操作**：
  - 查看详情
  - 导出结果
  - 删除回测

### 2.6 个人资料模块

#### 2.6.1 资料查看
- **功能描述**：查看个人信息
- **显示字段**：
  - 用户名
  - 邮箱
  - 手机号
  - 地址
  - 个人简介
  - 总资产
  - 持仓数量
  - 用户 ID
  - 注册时间

#### 2.6.2 资料编辑
- **功能描述**：编辑个人信息
- **可编辑字段**：
  - 用户名
  - 邮箱
  - 手机号
  - 地址
  - 个人简介
- **验证规则**：
  - 邮箱格式有效
  - 手机号格式有效（11 位数字）
  - 用户名不能为空

#### 2.6.3 账户统计
- **功能描述**：显示账户统计信息
- **统计内容**：
  - 总资产
  - 持仓数量
  - 用户 ID

## 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 3 秒 |
| API 响应时间 | < 1 秒 |
| 数据库查询时间 | < 500 ms |
| 并发用户数 | 1000+ |
| 数据更新频率 | 5 秒 |

### 3.2 可用性需求

- **系统可用性**：99.5% 以上
- **故障恢复时间**：< 5 分钟
- **数据备份**：每天备份一次
- **灾难恢复**：支持数据恢复

- **身份认证**：用户名密码认证
- **会话管理**：Token 机制
- **数据加密**：HTTPS 传输
- **访问控制**：基于角色的权限控制
- **审计日志**：记录重要操作
- **SQL 注入防护**：参数化查询
- **XSS 防护**：输入验证和输出编码

### 3.4 可扩展性需求

- **模块化设计**：支持功能扩展
- **API 版本管理**：支持多个 API 版本
- **数据库扩展**：支持数据分片
- **缓存机制**：支持缓存层

### 3.5 兼容性需求

- **浏览器支持**：Chrome、Firefox、Safari、Edge 最新版本
- **操作系统**：Windows、macOS、Linux
- **移动设备**：iOS、Android（响应式设计）
- **分辨率**：支持 1024×768 及以上

### 3.6 可维护性需求

- **代码质量**：遵循编码规范
- **文档完整性**：API 文档、用户手册
- **日志记录**：详细的系统日志
- **监控告警**：性能监控和告警

## 业务流程

### 5.1 用户登录流程

```
开始
  ↓
输入用户名和密码
  ↓
验证凭证
  ├─ 成功 → 生成 Token → 返回用户信息 → 跳转到首页
  └─ 失败 → 显示错误提示 → 返回登录页
  ↓
结束
```

### 5.2 交易流程

```
开始
  ↓
选择股票
  ↓
输入交易参数（类型、数量、价格）
  ↓
验证参数
  ├─ 失败 → 显示错误提示 → 返回
  └─ 成功 ↓
创建订单
  ↓
显示订单确认
  ↓
用户确认
  ├─ 取消 → 删除订单 → 返回
  └─ 确认 ↓
执行订单
  ↓
更新账户信息
  ├─ 买入 → 增加持仓，减少余额
  └─ 卖出 → 减少持仓，增加余额
  ↓
生成交易记录
  ↓
显示成功提示
  ↓
结束
```

### 5.3 回测流程

```
开始
  ↓
输入回测参数
  ↓
验证参数
  ├─ 失败 → 显示错误提示 → 返回
  └─ 成功 ↓
获取历史数据
  ↓
应用策略规则
  ↓
模拟交易执行
  ↓
计算性能指标
  ↓
生成回测报告
  ↓
保存回测结果
  ↓
显示结果
  ↓
结束
```

### 5.4 持仓管理流程

```
开始
  ↓
查看持仓列表
  ↓
选择持仓
  ↓
选择操作（加仓/减仓）
  ↓
输入数量
  ↓
验证数量
  ├─ 失败 → 显示错误提示 → 返回
  └─ 成功 ↓
更新持仓
  ↓
重新计算成本价
  ↓
更新统计信息
  ↓
显示成功提示
  ↓
结束
```

## 约束与假设

### 8.1 约束

- **技术约束**：
  - 后端使用 Python Flask
  - 前端使用 Vue 3
  - 数据库使用 SQLite
  - 部署使用 Docker

- **时间约束**：
  - 项目周期：3 个月
  - 开发阶段：6 周
  - 测试阶段：2 周
  - 部署阶段：1 周

- **资源约束**：
  - 开发人员：3 人
  - 测试人员：1 人
  - 产品经理：1 人

### 8.2 假设

- 用户具有基本的计算机操作能力
- 网络连接稳定
- 数据准确性由数据源保证
- 用户遵守系统使用规则
- 不涉及真实资金交易
